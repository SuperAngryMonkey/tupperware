# Tupperware v0.1.2 Release Notes

## Bug fixes

- **JavaScript escape bug in web UI** — clone button silently failed because of corrupted escape sequences in JS strings. Replaced with safer `String.fromCharCode(10)` for newlines and removed destructuring/regex patterns that didn't render cleanly through Flask's template engine.
- **Script name mismatch in installer** — `install.sh` was dropping scripts as `build-tailscale-template.sh` and `new-tailscale-ct.sh`, but the Flask app expected `tupperware-build-template` and `tupperware-new`. Now installs with consistent `tupperware-*` names everywhere. Existing installs with the legacy names are detected and cleaned up automatically on re-run.

## New features

- **Inventory panel** — below the clone form, a read-only table of all LXC containers on the host showing VMID, hostname, status, CPU/memory/disk, LAN IP, tailnet IP, tags, and notes (from Proxmox's `description` field).
- **Refresh button** on the inventory panel to re-query container state.

## Upgrading from v0.1.0 or v0.1.1

Run the installers again — they're idempotent and will clean up the old script names:

```bash
curl -fsSL https://raw.githubusercontent.com/SuperAngryMonkey/tupperware/main/scripts/install.sh | bash
curl -fsSL https://raw.githubusercontent.com/SuperAngryMonkey/tupperware/main/scripts/install-webui.sh | bash
```

The web UI service is restarted automatically. Hard-refresh your browser (Cmd+Shift+R) to pick up the new HTML.

## Setting container notes

To add a description that appears in the inventory:

```bash
pct set <vmid> -description "Your notes here"
```

Or use the Proxmox web UI: Container → Notes tab.
