# Tupperware v0.1.3 Release Notes

## New features

- **Storage backend selection** — when a Proxmox host has multiple storage backends (e.g., added a second drive as `local-zfs`, `data-nvme`, `data-hdd`), Tupperware now lets you pick which one to clone into.
  - **Web UI**: new "Storage Backend" dropdown in the clone form, populated by querying `pvesm status -content rootdir` on page load. Shows the storage name and type.
  - **CLI**: new `--storage <name>` flag for `tupperware-new`. Existing `STORAGE=<name>` env var still works.
- **Inventory shows storage** — the inventory table has a new "STORAGE" column showing which backend each container's rootfs lives on. Useful for spotting which containers are on which drive.
- **Storage validation** — before cloning, the CLI verifies the chosen storage actually exists and supports container content (rootdir). If not, fails fast with the list of valid options.

## Defaults

- Default storage remains `local-lvm` for backward compatibility. Existing installs continue to work without changes.

## Upgrading from v0.1.2

```bash
curl -fsSL https://raw.githubusercontent.com/SuperAngryMonkey/tupperware/main/scripts/install.sh | bash
curl -fsSL https://raw.githubusercontent.com/SuperAngryMonkey/tupperware/main/scripts/install-webui.sh | bash
```

Hard-refresh your browser (Cmd+Shift+R) after the web UI restarts.

## Usage examples

### Web UI

The clone form has a new dropdown. Pick your storage, click Clone & Join Tailnet.

### CLI

```bash
# Default storage (local-lvm)
tupperware-new 201 lab-lxc-01

# Specific storage
tupperware-new 202 db-cache --storage local-zfs

# Or via env var
STORAGE=data-nvme tupperware-new 203 fast-app

# List available storage backends
pvesm status -content rootdir
```

## Note on storage backends

Only storage backends that support **container content** (`rootdir`) appear in the dropdown. If you have storage configured but it doesn't show up, check that it's enabled for container content in `/etc/pve/storage.cfg` or via the Proxmox web UI: **Datacenter → Storage → Edit → Content → check "Container"**.
